## ğŸ“– What I Learned

### ğŸ”¹ Linear Search
- Checks elements **one by one** from start to end
- Works on **both sorted and unsorted arrays**
- Simple to implement
- Suitable for small datasets

**Time Complexity:**
- Best Case: `O(1)`
- Average Case: `O(n)`
- Worst Case: `O(n)`

**Real-life Example:**  
Searching a name in an unsorted attendance list.

---

### ğŸ”¹ Binary Search
- Works only on **sorted arrays**
- Divides the search space into halves
- Much faster than Linear Search for large datasets

**Time Complexity:**
- Best Case: `O(1)`
- Average Case: `O(log n)`
- Worst Case: `O(log n)`

**Real-life Example:**  
Searching a word in a dictionary.
